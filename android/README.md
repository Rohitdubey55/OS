# `android/` — Capacitor Android wrapper

Native Android side of PersonalOS. The web shell lives in `../web/` and is copied here by `npx cap sync android` into `app/src/main/assets/public/`. Everything in this folder is **Android-only** — Java, XML, Gradle, drawables, manifest.

## Build / install

```bash
# From repo root, NOT from android/
npm run sync        # rebuilds www/ AND runs npx cap sync android
npm run android     # also opens Android Studio
```

Or manually:
```bash
cd android
./gradlew installDebug         # build + install debug APK
./gradlew assembleRelease      # build a signed release (needs signing config)
```

To uninstall (use when widgets fail to add — Android caches widget metadata):
```bash
adb uninstall com.personal.os
```

## Native plugin pattern

Custom Capacitor plugins live in `app/src/main/java/com/personal/os/`. Pattern:

```java
@CapacitorPlugin(name = "MyPlugin")
public class MyPlugin extends Plugin {
    @PluginMethod
    public void myMethod(PluginCall call) {
        String arg = call.getString("arg");
        JSObject result = new JSObject();
        result.put("ok", true);
        call.resolve(result);
    }
}
```

Then register in `MainActivity.onCreate` **before** `super.onCreate`:
```java
registerPlugin(MyPlugin.class);
```

And expose to JS in `web/capacitor-init.js`:
```js
Capacitor.Plugins.MyPlugin = registerPlugin('MyPlugin');
window.MyPlugin = Capacitor.Plugins.MyPlugin;
```

**Always expose under BOTH `window.X` and `Capacitor.Plugins.X`** — `Capacitor.Plugins` is sometimes managed by core and direct assignment can be silently dropped. The `window.X` global is the reliable lookup.

## Widget anatomy

A widget = (1) `<receiver>` in manifest + (2) provider info XML + (3) layout XML + (4) `AppWidgetProvider` Java class. To add a new widget:

1. **Layout** in `res/layout/widget_my.xml`. **RemoteViews only.** Allowed tags: `LinearLayout, FrameLayout, RelativeLayout, GridLayout, AdapterViewFlipper, GridView, ListView, StackView, ViewFlipper, AnalogClock, Button, Chronometer, ImageButton, ImageView, ProgressBar, TextView, ViewStub`. **`<View>` is NOT allowed on Samsung One UI** — use `<FrameLayout>` for spacers and colored dots.
2. **Provider info** in `res/xml/widget_my_info.xml` (size, refresh period, preview layout). Do not set `android:configure=""` to an empty string — that crashes the picker. Just omit it.
3. **Java class** extends `AppWidgetProvider`. Override `onUpdate`. Read data from `WidgetBridgePlugin.PREFS_NAME` SharedPreferences.
4. **Receiver** in `AndroidManifest.xml`:
   ```xml
   <receiver
       android:name="com.personal.os.widgets.MyWidget"
       android:label="@string/widget_my_label"
       android:exported="true">       <!-- MUST be true for Samsung launcher -->
       <intent-filter>
           <action android:name="android.appwidget.action.APPWIDGET_UPDATE" />
       </intent-filter>
       <meta-data android:name="android.appwidget.provider"
                  android:resource="@xml/widget_my_info" />
   </receiver>
   ```
5. **Keep rule** in `proguard-rules.pro` (already covers `com.personal.os.widgets.**`).
6. **Add to broadcast list** in `WidgetBridgePlugin.WIDGET_CLASSES[]` so JS-pushed data triggers a refresh.

## Theme / system bars

- `styles.xml` `AppTheme.NoActionBar` has `windowOptOutEdgeToEdgeEnforcement="true"` (Android 15 opt-out)
- `MainActivity.onCreate` calls `WindowCompat.setDecorFitsSystemWindows(window, true)` as Java-level fallback
- Status bar color is controlled at runtime by `web/main.js` `syncNativeChromeToTheme()` via `StatusBar.setBackgroundColor`

If status bar overlap returns, check both the theme attr and the WindowCompat call — both must be in place.

## Common gotchas

- **"Couldn't add widget":** usually one of (1) Samsung One UI rejecting `<View>` — use `<FrameLayout>`. (2) `android:configure=""` — remove it. (3) R8 stripping the widget class — keep rule must be in `proguard-rules.pro`. (4) `android:exported="false"` on the receiver — flip to true.
- **Widget data empty:** check Logcat for `WidgetBridge` and `WidgetClassName` tags. JS-side log: `[WidgetBridge] Android: plugin returned {ok: true, bytes: NNN}`. If that fires but the widget stays empty, the broadcast in `WidgetBridgePlugin.triggerWidgetUpdate` isn't reaching the receiver.
- **Capacitor sync overwrites things:** `capacitor.build.gradle` and `capacitor.settings.gradle` regenerate on every `cap sync`. Never edit them. Everything else is yours.
- **R8 release builds:** debug builds don't minify. If something works in debug and breaks in release, check `proguard-rules.pro` for missing keep rules.
